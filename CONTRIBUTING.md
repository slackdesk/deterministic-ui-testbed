# Contributing

## Development setup

- Use Node (npm) for Playwright tests, and/or Python for system-Chromium workflows.
- Keep tests deterministic: prefer measurable signals (status, counts, timings).

## Pull requests

- Keep PRs small and focused.
- Include a brief description of what changed and why.
- Add/adjust tests when behavior changes.

## Code style

- Favor readability.
- Avoid flaky checks.
